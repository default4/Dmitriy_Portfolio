PROJECTS

Credit Card Fraud Detection

June 7, 2023

This project aims to build a model to detect credit card fraud using a dataset of credit card transactions. Each transaction is labeled as either fraudulent or non-fraudulent.

Dataset

The dataset contains 284,807 transactions, each represented by 31 columns:

  • Time: Seconds elapsed between this transaction and the first transaction in the dataset.
  • V1 to V28: PCA transformed features.
  • Amount: The transaction amount.
  • Class: Target variable (1 represents a fraudulent transaction, and 0 represents a non-fraudulent transaction).

Steps

1. Data Exploration & Preprocessing

  • Explored the structure of the dataset and distribution of fraudulent and non-fraudulent transactions.
  • Visualized the distribution of transaction amount and time, as well as the correlation between features.
  • Found that the dataset is highly imbalanced with a majority of non-fraudulent transactions.

Class Distribution: The bar plot shows a highly imbalanced distribution of the target variable ‘Class’, with the majority of transactions being non-fraudulent (Class 0) and a very small number being fraudulent (Class 1).

Transaction Amount Distribution: The histogram of the ‘Amount’ feature shows a highly right-skewed distribution, indicating that most transactions are of low monetary value, with a few transactions having very high amounts.

Transaction Time Distribution: The histogram of the ‘Time’ feature shows a bimodal distribution, indicating two periods of high transaction frequency within the given time frame.

Correlation Matrix: The heatmap of the correlation matrix of features does not show significant correlations between features, except for some features showing correlation with ‘Amount’.

2. Feature Engineering

  • Standardized the ‘Amount’ feature.
  • Explored different strategies to handle the imbalanced dataset:
    • Baseline Model (without handling imbalance)
    • Oversampling (SMOTE) - not implemented due to library limitations.
    • Undersampling
    • Adjusting Class Weights

3. Model Selection & Training

  • Split the dataset into training, validation, and test sets.
  • Trained a Logistic Regression model using different strategies for handling class imbalance.
  • Evaluated the model’s performance on the validation set using metrics such as accuracy, precision, recall, F1-score, and AUC-ROC score.

4. Visualization of Model Results

  • Visualized the results using:
    • Confusion Matrix: To evaluate the number of True Positives, True Negatives, False Positives, and False Negatives.
  • ROC Curve: To illustrate the diagnostic ability of the classifier.
  • Precision-Recall Curve: To depict the trade-off between precision and recall for different threshold values.

5. Model Evaluation

  • Compared the performances of the models trained using different strategies.
  • Chose the Baseline Model for further evaluation as it demonstrated a good balance between precision and recall.
  • Evaluated the chosen model on the test set and visualized the results using a confusion matrix, ROC curve, and Precision-Recall curve.

Results

The Baseline Model achieved the following performance on the test set:

  • Accuracy: 99.93%
  • Precision (Fraudulent Class): 92.00%
  • Recall (Fraudulent Class): 62.16%
  • F1-score (Fraudulent Class): 74.19%
  • AUC-ROC Score: 0.81

The model demonstrated a good ability to distinguish between fraudulent and non-fraudulent transactions and maintained a balance between precision and recall.
